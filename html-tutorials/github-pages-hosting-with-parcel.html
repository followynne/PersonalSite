Use Parcel to bundle libraries and publish your site with GH-Pages Hosting

Hi! If you are here, you're searching for instructions to publish a site on the free hosting servers offered by GitHub, the popular site to host git remote repository, using Parcel as a bundler for all the packages/libraries you used in your site (ex: jQuery, Bootstrap...) :) Look no more, as here I've written a simple guide to use Parcel to bundle all those libraries and push the built pages on the gh-pages branch of your github repo.
For this tutorial, you'll need to have installed Node.Js, npm (the Javascript package manager we'll use to install libraries and use parcel) and well, of course, Parcel. For those operations you can find on the net loads of information about.
Please, a note: $ identifies code that goes in a Shell/Terminal (CMD) - when you paste the code lines in the shell, leave out the starting $. You can find all the instructions and commands inside a code tag.
Last reminder: if you want, you can find by clicking Site_Example_Code the entire code of a basic site that was created using this tutorial. You can check it live at Repo_Model.

Okay, let's start by assuming you have created your Hello World Site. You've created all the folders for your files, you've got your nice index.html ready and working, with the stylesheet and scripts correctly put (please bear in mind it would be better to write paths for those files with the path starting with a full stop, when referring to folder/files on lower levels than the index. Ex: ./path/to/folder).
You uploaded it on GitHub and it looks neat.
Let's assume you used Bootstrap and JQuery. You installed the packages you need by using npm in the Terminal:
$ cd "your_folder_path"
$ npm init
$ npm install bootstrap
$ npm install popper.js
$ npm install jquery
At the start of your main javascript file you required the libraries: 
import $ from 'jquery'
import 'popper.js'
import 'bootstrap/dist/css/bootstrap.min.css'
import 'bootstrap/dist/js/bootstrap'
and in the same file you wrote a simple function to test the import: 
$(document).ready(()=>{
$('body').html(
    'Hello World, if you see this message Parcel Bundler and Github Hosting are working
    nicely!')
})
You ran parcel ($ parcel index.html) and magic, it worked! Now we want to publish it using GH-Pages Hosting.

First thing, in the .gitignore let's add those lines: 
node_modules/
dist
.cache$ parcel index.html
then we need to install a new package from npm that will take care of creating the gh-pages branch on your remote repo: 
$ npm install gh-pages
Let's create a GH-Pages branch and push it to your remote repo: 
$ git branch gh-pages
$ git checkout gh-pages
$ git add .
$ git commit -m "first commit"
$ git push origin gh-pages
$ git checkout master
$ npm install gh-pages
We're back on the main branch, with our gh-pages running (you can check that the page is online in your online repo, by going to Settings).
Now we need to open the package.json to add this lines before "dependencies":{
"scripts": {
"start": "parcel index.html",
"predeploy": "rm -rf dist && parcel build index.html --public-url "paste here your ENTIRE REPO URL, comprehensive of http/https part"",
"deploy": "gh-pages -d dist"
}
What are we doing? We are creating a script for npm that will execute those commands:
  1. Pre-Deploy: it will delete the previous dist folder (the folder where are located the processed parcel files) + it will build the site from your entry point;
  2. Deploy: it will use gh-pages package to re-create the gh-pages branch based on the dist folder and it will "commit&push" it to your remote repo
  3. Start: it will start parcel, to let you check what's been built.
Please Note: if you have a CNAME file, containing the personalized url of your site, you !MUST! substitute the "predeploy" line in the last snippet with:
"predeploy": "rm -rf dist && parcel build index.html --public-url "paste here your PERSONAL url, comprehensive of http/https part" && cp CNAME ./dist"
That's needed because this instruction will make Parcel build your page on the specified url and it will copy the CPNAME file in the dist folder - this way GitHub will understand which custom domain you're using. :)
Now the job's easy: every time you have a new site build ready to deploy, you type this command in Terminal: 
$ npm run deploy
Parcel will run, will build the site and deploy it to GitHub with all the dependencies you used and are needed by your scripts. Fast, easy, no CDN used :)
